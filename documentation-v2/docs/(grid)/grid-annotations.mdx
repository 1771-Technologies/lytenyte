Annotations overlay arbitrary React content on top of the grid. Each annotation has two parts:

- An **anchor** that describes where to position it.
- A renderer function that returns what content to display at the anchor position.

LyteNyte Grid accepts an array of annotations via the `annotations` property on the grid.

The demo below shows a basic annotation that highlights a particular area of the grid using
a dashed rectangular area.

::demo[Grid Annotations="./demos/grid-annotations"]

The content rendered by an annotation can be anything your application requires. LyteNyte Grid provides
the positioning system, your application provides the content to render.

## Anchor Types

Three anchor kinds are supported. Set the `kind` field to choose one.

### Range

A range annotation covers a rectangular region of data cells. `rowStart`/`rowEnd` and
`colStart`/`colEnd` accept either a numeric index or a string ID.

```tsx
anchor: {
  kind: "range",
  rowStart: 0,        // row index or row ID
  rowEnd: 5,          // row index or row ID
  colStart: "revenue", // column index or column ID
  colEnd: "profit",    // column index or column ID
}
```

LyteNyte Grid automatically splits the annotation across pinned sections (start-pinned, center, and
end-pinned columns) and across top-pinned, center, and bottom-pinned rows. This means a single
range annotation renders correctly even when it spans both pinned and unpinned columns or rows.

### Header

A header annotation covers a span of columns in the column header area.

```tsx
anchor: {
  kind: "header",
  colStart: "revenue", // column index or column ID
  colEnd: "profit",    // column index or column ID
}
```

Header annotations are split across the start-pinned, center, and end-pinned header sections, just
like range annotations.

### Point

A point annotation places content at an absolute `(x, y)` coordinate in the scrollable data area.
The values are pixel offsets from the top-left corner of the first non-pinned row in the center
scroll area, the same coordinate space that the virtual row layout uses.

```tsx
anchor: {
  kind: "point",
  x: 400,
  y: 300,
}
```

Point annotations are not split, they render in the center scroll section only.

## The Render Function

The `render` function receives `{ api }` and returns a React node. The annotation
container is already positioned and sized to match the anchor. Use `position: "absolute"` with
`inset: 0` to fill the container, or use any CSS positioning approach to place content within it.

```tsx
render: ({ api }) => {
  // api gives you access to the full grid API
  return (
    <div
      style={{
        position: "absolute",
        inset: 0,
        border: "2px solid #ef4444",
        background: "rgba(239, 68, 68, 0.08)",
        boxSizing: "border-box",
        pointerEvents: "none",
      }}
    />
  );
},
```

By default, the annotation container has `pointer-events: none` applied. If your render content
needs to respond to mouse events — for example, a clickable badge or tooltip trigger, set
`pointerEvents: "auto"` on the element that should receive events.

## Copy Marching Ants

The [marching ants](https://en.wikipedia.org/wiki/Marching_ants) effect is an animated dashed border
that signals a copied selection, familiar from spreadsheet applications like Excel and Google Sheets.
Annotations make it straightforward to replicate this pattern in LyteNyte Grid.

The demo below supports the standard clipboard shortcuts. Select a range of cells, then:

- **Ctrl/Cmd + C:** copies the selection to the clipboard and shows the marching ants border.
- **Ctrl/Cmd + X:** same as copy, but pasting will also clear the original cells (cut semantics).
- **Ctrl/Cmd + V:** pastes at the active cell. If the clipboard still holds the original copied
  content and the operation was a cut, the source cells are cleared. If you copied something else
  between copying and pasting, the ants are dismissed and source cells are left untouched.
- **Escape:** dismisses the ants without pasting.

The ants persist while focus stays within the grid, including when you click other cells. They are
cleared when focus moves to a different part of the page, but not when you switch to another
application, so you can copy, Alt-Tab to check something, and return to paste without losing the
indicator.

The animation itself is an SVG `<rect>` with `stroke-dasharray` driven by a CSS `@keyframes` rule
that increments `stroke-dashoffset`, creating the illusion of movement along the border.
