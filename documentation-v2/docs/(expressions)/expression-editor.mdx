## Overview

`ExpressionEditor` is a React compound component that wraps a `<textarea>` with two features
layered on top:

1. **Syntax highlighting**: the input is tokenized on every keystroke and each token is rendered
   with a type-specific colour via CSS variables.
2. **Completions**: an optional floating popover shows context-aware suggestions as the user types,
   with full keyboard navigation.

Both features are opt-in. You can use the editor as a pure syntax-highlighted input without any
completion UI.

## Installation

Import the stylesheet once before rendering `ExpressionEditor`:

```ts
import "@1771technologies/lytenyte-pro/expression-editor.css";
```

## Minimal Setup

The only required props are `value`, `onChange`, and `tokenize`. Rendering `ExpressionEditor.Root`
with no children gives you a fully functional syntax-highlighted input with no completion UI:

```tsx
import {
  Evaluator,
  ExpressionEditor,
  standardPlugins,
} from "@1771technologies/lytenyte-pro/expressions";
import { useCallback, useState } from "react";

const evaluator = new Evaluator(standardPlugins);

function MyEditor() {
  const [value, setValue] = useState("");
  const tokenize = useCallback((s: string) => evaluator.tokensSafe(s, true), []);

  return (
    <div data-ln-input="true" style={{ height: 40 }}>
      <ExpressionEditor.Root value={value} onChange={setValue} tokenize={tokenize} />
    </div>
  );
}
```

Use `tokensSafe` rather than `tokens` so that the editor continues to render partial highlights
even when the expression is syntactically incomplete.

## Compound Component Structure

`ExpressionEditor` is a compound component. Compose it from its sub-components to control
exactly which parts of the completion UI are rendered:

| Sub-component                         | Element  | Role                                                                          |
| ------------------------------------- | -------- | ----------------------------------------------------------------------------- |
| `ExpressionEditor.Root`               | `<div>`  | The editor itself. Manages state and passes context to children.              |
| `ExpressionEditor.CompletionPopover`  | `<div>`  | A `popover="manual"` panel positioned at the cursor. Shows when completions are ready. |
| `ExpressionEditor.CompletionList`     | `<ul>`   | Renders the suggestion list via a render prop `({ items, loading }) => ReactNode`. |
| `ExpressionEditor.CompletionListItem` | `<li>`   | Renders one suggestion. Handles `mousedown` selection and `aria-selected` state. |

All sub-components accept `className`, `style`, and any valid HTML attribute for their underlying
element. Omit `CompletionPopover` entirely to disable the completion UI.

The full structure looks like this:

```tsx
<ExpressionEditor.Root value={value} onChange={setValue} tokenize={tokenize} completionProvider={provider}>
  <ExpressionEditor.CompletionPopover>
    <ExpressionEditor.CompletionList>
      {({ items }) =>
        items.map((item, index) => (
          <ExpressionEditor.CompletionListItem key={item.id} item={item} index={index}>
            {item.label}
          </ExpressionEditor.CompletionListItem>
        ))
      }
    </ExpressionEditor.CompletionList>
  </ExpressionEditor.CompletionPopover>
</ExpressionEditor.Root>
```

## The Completion System

### Providing Completions

Pass a `completionProvider` function to `ExpressionEditor.Root`. It receives the current token
array and the cursor position (the index of the token under the cursor), and returns either a
synchronous array or a `Promise`:

```ts
type CompletionProvider = (
  tokens: Token[],
  cursorPosition: number,
) => CompletionItem[] | Promise<CompletionItem[]>;
```

Each item has three fields:

```ts
type CompletionItem = {
  label: string; // text shown in the list
  kind: string;  // semantic kind, drives icons/colours in your UI ("function", "string", "number", …)
  id: string;    // unique key for React
};
```

### Using `createCompletionProvider`

`createCompletionProvider` builds a provider from a context object. Pass it the same object you
pass to `evaluator.run`:

```ts
import { createCompletionProvider } from "@1771technologies/lytenyte-pro/expressions";

const context = {
  user: { firstName: "John", lastName: "Smith" },
  age: 23,
  fullName: () => context.user.firstName + " " + context.user.lastName,
};

const provider = createCompletionProvider(context);
```

The provider inspects the token stream before the cursor to pick the right set of suggestions:

| Context | Completions shown |
| ------- | ----------------- |
| Top level (empty or after an operator) | All keys of the context object |
| `identifier.` | Properties of that identifier resolved through the context |
| `identifier.nested.` | Properties resolved by walking the full path |
| `"string".` | All built-in `String` methods (`length`, `includes`, `slice`, …) |
| `[array].` | All built-in `Array` methods (`map`, `filter`, `reduce`, …) |
| Anything else | No completions |

### Completion Trigger and Filtering

Completions open in two ways:

- **Automatic**: typing `.` immediately triggers a completion fetch for the property access context.
- **Manual**: pressing `Ctrl+Space` (or `⌘+Space` on macOS) opens completions from wherever the cursor is.

Once the popover is open, the list is filtered in real time as the user continues typing. The filter
is a simple prefix match on `item.label`, so typing `to` narrows a string method list to
`toLowerCase`, `toUpperCase`, and `toString`.

Completions close when the user accepts a suggestion, presses `Escape`, or moves focus away from
the editor.

### Writing a Custom Provider

Implement the `CompletionProvider` signature directly when you need domain-specific suggestions that
`createCompletionProvider` does not cover:

```ts
import type { CompletionItem } from "@1771technologies/lytenyte-pro/expressions";
import type { Token } from "@1771technologies/lytenyte-pro/types";

const FIELDS: CompletionItem[] = [
  { label: "Revenue", kind: "number", id: "Revenue" },
  { label: "Region",  kind: "string", id: "Region"  },
  { label: "InStock", kind: "boolean", id: "InStock" },
];

const FUNCTIONS: CompletionItem[] = [
  { label: "round", kind: "function", id: "round" },
  { label: "abs",   kind: "function", id: "abs"   },
];

function myProvider(tokens: Token[], cursorPosition: number): CompletionItem[] {
  const relevant = tokens.filter(
    (t) => t.end <= cursorPosition && t.type !== "EOF" && t.type !== "Whitespace",
  );

  // Skip the word being typed
  let i = relevant.length - 1;
  if (relevant[i]?.type === "Identifier") i--;

  const prev = relevant[i];

  // After a dot, return nothing (our fields have no sub-properties)
  if (prev?.type === "Punctuation" && prev.value === ".") return [];

  // At the top level, offer fields and formula functions
  return [...FUNCTIONS, ...FIELDS];
}
```

## Customizing Syntax Highlighting

The default highlighter maps each token type to a CSS variable colour. Override the `highlight`
prop to replace it entirely:

```tsx
import type { Token } from "@1771technologies/lytenyte-pro/types";
import type { ReactNode } from "react";

function MyHighlighter({ token }: { token: Token }): ReactNode {
  if (token.type === "Number") return <span style={{ color: "cornflowerblue" }}>{token.value}</span>;
  if (token.type === "String") return <span style={{ color: "mediumseagreen" }}>{token.value}</span>;
  if (token.type === "ExpressionError") return <span style={{ textDecoration: "wavy underline red" }}>{token.value}</span>;
  return <span>{token.value}</span>;
}

<ExpressionEditor.Root tokenize={tokenize} highlight={MyHighlighter} … />
```

### CSS Variables

The default highlighter reads these CSS custom properties. Override them in your theme to change the
colour scheme without replacing the highlighter:

| Variable               | Token types                                              |
| ---------------------- | -------------------------------------------------------- |
| `--ln-expr-number`     | `Number`                                                 |
| `--ln-expr-string`     | `String`, `TemplateLiteral`                              |
| `--ln-expr-operator`   | `Operator`, `Punctuation`, `Spread`, `Pipe`, `Arrow`     |
| `--ln-expr-identifier` | `Identifier`                                             |
| `--ln-expr-error`      | `ExpressionError`                                        |
| `--ln-expr-punctuation`| `Unparsed`                                               |

## Prop Reference

### `ExpressionEditor.Root`

| Prop                 | Type                                                                  | Required | Description                                                                   |
| -------------------- | --------------------------------------------------------------------- | -------- | ----------------------------------------------------------------------------- |
| `value`              | `string`                                                              | Yes      | The current expression string (controlled).                                   |
| `onChange`           | `(value: string) => void`                                             | Yes      | Called whenever the user edits the expression.                                |
| `tokenize`           | `(value: string) => Token[]`                                          | Yes      | Converts the expression string into tokens for highlighting. Use `tokensSafe` to avoid throwing on incomplete input. |
| `highlight`          | `(props: { token: Token }) => ReactNode`                              | No       | Custom token renderer. Defaults to the built-in CSS-variable highlighter.     |
| `completionProvider` | `(tokens: Token[], cursorPosition: number) => CompletionItem[] \| Promise<CompletionItem[]>` | No | Completion function. Omit to disable the completion system entirely. |
| `placeholder`        | `string`                                                              | No       | Placeholder text shown when `value` is empty.                                 |
| `disabled`           | `boolean`                                                             | No       | Disables the underlying `<textarea>`.                                         |
| `readOnly`           | `boolean`                                                             | No       | Makes the editor read-only.                                                   |
| `className`          | `string`                                                              | No       | CSS class applied to the outer `<div>`.                                       |
| `style`              | `CSSProperties`                                                       | No       | Inline styles applied to the outer `<div>`.                                   |

### `ExpressionEditor.CompletionPopover`

Accepts all `<div>` HTML attributes. Positioned automatically at the cursor using the native
[Popover API](https://developer.mozilla.org/en-US/docs/Web/API/Popover_API). The popover opens
and closes based on completion state; you do not need to manage this yourself.

### `ExpressionEditor.CompletionList`

Accepts all `<ul>` HTML attributes plus a render prop `children`:

```ts
children?: (props: { items: CompletionItem[]; loading: boolean }) => ReactNode
```

`loading` is `true` while an async provider is still resolving. Use it to show a spinner if your
provider makes a network request.

### `ExpressionEditor.CompletionListItem`

| Prop    | Type             | Required | Description                                                                  |
| ------- | ---------------- | -------- | ---------------------------------------------------------------------------- |
| `item`  | `CompletionItem` | Yes      | The completion item to render.                                               |
| `index` | `number`         | Yes      | The item's position in the list. Used to compute `aria-selected` state.      |

Also accepts all `<li>` HTML attributes. Target the selected state with `[aria-selected="true"]`
or `aria-selected` in Tailwind.

## Keyboard Reference

| Key                    | When completion is open         | When completion is closed |
| ---------------------- | ------------------------------- | ------------------------- |
| `↑` / `↓`             | Navigate up / down through list | —                         |
| `Enter` or `Tab`       | Accept selected suggestion      | Insert newline (multiline only) |
| `Escape`               | Dismiss completions             | —                         |
| `Ctrl+Space` / `⌘+Space` | —                             | Open completions manually |
| `.`                    | —                               | Auto-trigger completions for the property access context |
