## Grid Context Menu

Use the `events` property on the grid to set a `contextmenu` event listener on the part
of the grid that should open a contextual menu.

The demo below shows how to add a context menu to grid cells. Right-click on any cell to
open the context menu, then select an action to perform.

::demo[Context Menu="./demos/context-menu"]

The demo uses the event code shown below. The grid registers `contextmenu` event handlers
for both header cells and row cells. Because the browser displays its own context menu on
right-click, the handlers call `preventDefault` to suppress the native behavior.

Each event handler uses the `virtualFromXY` utility function provided by LyteNyte Grid to
construct a virtual element. The handler then assigns this virtual element to the anchor
state. The LyteNyte Grid menu can use the virtual element as a positioning anchor, which
eliminates the need for a menu trigger.

```tsx
<Grid
  rowHeight={50}
  columns={columns}
  onColumnsChange={setColumns}
  ref={setApi}
  rowSource={ds}
  //!next 22
  events={useMemo<Grid.Events<GridSpec>>(() => {
    return {
      headerCell: {
        contextMenu: ({ event, column }) => {
          event.preventDefault();
          event.stopPropagation();
          setAnchor(virtualFromXY(event.clientX, event.clientY));

          setMenu({ column, row: null });
        },
      },
      cell: {
        contextMenu: ({ event, row, column }) => {
          event.preventDefault();
          event.stopPropagation();
          setAnchor(virtualFromXY(event.clientX, event.clientY));

          setMenu({ row, column });
        },
      },
    };
  }, [])}
/>
```
